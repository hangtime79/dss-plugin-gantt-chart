# Release Notes: v0.9.1

**Release Date:** 2025-12-27
**Type:** Feature
**Branch:** `feature/v0.9.1-dark-mode`

---

## Summary

Completes the dark mode implementation with comprehensive fixes for all remaining dark mode issues. Addresses 9 visual gaps identified during QA of v0.9.0, plus adds a new completion indicator feature showing checkmarks on 100% complete tasks.

---

## Changes

### Added
- **Completion indicator for 100% tasks**: SVG checkmark displayed on completed tasks
  - Centered in left portion of bar with label shifted right
  - Adaptive color (black on light bars, white on dark bars) using luminance calculation
  - Works across all color palettes (default tier, classic, pastel, Dataiku, dark)
- Dark theme zoom limit banner styling (gold/amber instead of yellow)
- Dark theme button hover/active states
- Dark theme default tier bar colors (dark slate blue)
- Dark theme pastel palette overrides (saturated versions)
- `_is_complete` flag added to task data for frontend detection

### Fixed
- Sticky header now uses CSS variable `var(--color-surface)` instead of hardcoded colors
- Header theme reapplication on view change
- Header text contrast improved (uses `--text-main` for better visibility)
- Floating year (`.current-upper`) background now respects dark theme
- Progress bar contrast improved with darker overlay (`rgba(0,0,0,0.35)`)
