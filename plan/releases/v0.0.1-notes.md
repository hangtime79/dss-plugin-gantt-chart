# Release Notes: v0.0.1

**Release Date:** 2025-12-18
**Type:** Initial Release

---

## Summary

First implementation of the Gantt Chart plugin for Dataiku DSS. This release delivers a fully functional Gantt chart visualization that appears in the Charts tab, using the bundled Frappe Gantt library for offline operation.

---

## Features Delivered

### Core Functionality
- Interactive Gantt chart visualization in Charts tab → Other section
- Task bars positioned by start/end dates
- Task dependencies rendered as connecting arrows
- Progress tracking with filled progress bars
- Color coding by categorical column (12-color palette)

### Configuration Options
19 parameters across 5 sections:
- **Data Columns (7):** Task ID, Name, Start, End, Progress, Dependencies, Color By
- **View Settings (3):** Default view mode, View mode dropdown, Scroll position
- **Appearance (4):** Bar height, Corner radius, Column width, Padding
- **Behavior (4):** Read-only, Popup trigger, Today button, Weekend highlighting
- **Performance (1):** Max tasks limit

### View Modes
7 time scales: Hour, Quarter Day, Half Day, Day, Week, Month, Year

### Edge Case Handling
- Null/empty task names → Generated names ("Task 0", "Task 1")
- Null task IDs → Generated IDs ("task_0", "task_1")
- Duplicate task IDs → Suffixed IDs ("id_1", "id_2")
- Invalid dates → Rows skipped with metadata
- Start > End dates → Rows skipped
- Circular dependencies → Detected and broken using DFS
- Missing dependency references → Invalid references removed

---

## Technical Details

### Architecture
- **Business Logic:** `python-lib/ganttchart/` (4 modules, DSS-independent)
- **Webapp:** Flask backend + Frappe Gantt frontend
- **Offline:** All resources bundled (no CDN dependencies)

### Code Statistics
- ~2,200 lines of code
- 77 unit tests (100% passing)
- 4 Python modules
- 5 webapp files

### API Endpoints
- `GET /get-tasks` - Returns transformed task data
- `GET /get-config` - Returns Frappe Gantt configuration

## Known Issues at Release

1. Backend used invalid `dataiku.get_datadir()` API call
2. Loading spinner could hang indefinitely
3. `dataiku.webappBackend` not automatically available

These issues were addressed in v0.0.2.
