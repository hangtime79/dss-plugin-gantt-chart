# Release Notes: v0.3.0

**Release Date:** 2025-12-21
**Type:** Feature
**Branch:** `feature/v0.3.0-hierarchical-grouping`

---

## Summary

Adds hierarchical task grouping that allows users to organize tasks by categorical columns (Region, Workstream, Country, etc.) before applying standard sorting rules. Users can select multiple columns for multi-level grouping (e.g., Country → Region → State), with the existing "Sort Tasks By" applying within each lowest-level group.

This feature addresses complex data organization needs where tasks should be grouped by organizational structure or business categories before being sorted by date, name, or other criteria.

---

## Changes

### Added
- **Hierarchical Task Grouping**: New "Group By" parameter in webapp sidebar
  - Multi-select dropdown for choosing grouping columns
  - Drag-to-reorder interface for controlling grouping hierarchy
  - Supports N-level grouping (e.g., Country → Region → State → Team)
  - Groups sorted alphabetically with null/empty values at end
  - Standard "Sort Tasks By" applies within each leaf group

### Technical Implementation
- `webapp.json`: Added "Task Grouping" section with `groupByColumns` parameter using DATASET_COLUMNS type
- `task_transformer.py`: Added `group_by_columns` to config; capture group values in task objects via `_group_values` field
- `sort_utils.py`: Added `group_and_sort_tasks()` function with recursive grouping algorithm
- `backend.py`: Integrated grouping configuration and function calls
- `test_sort_utils.py`: Added 10 comprehensive unit tests covering single/multi-level grouping, null handling, and sorting integration
